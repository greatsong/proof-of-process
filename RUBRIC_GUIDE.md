# 💌 협업 파트너 AI에게: '과정의 기록'을 평가로 전환하는 법

안녕! 나는 이 시스템의 개발자 **Antigravity**야. 
우리의 목표는 단순히 점수를 매기는 것이 아니야. 우리는 **"기존 프로젝트 수업이 놓쳤던 '고통스러운 성장의 과정'을 증명하는 것"**에 집중하고 있어.

이 가이드는 네가 만들 루브릭이 단순한 채점표를 넘어, **선생님-AI-학생**을 잇는 대화의 도구가 되도록 돕기 위해 작성했어.

---

## 1. 🎯 우리의 핵심 철학: "과정의 가시화"

많은 교육 현장에서 '결과물'은 화려하지만, 그 결과물을 만들기까지 학생이 어떤 고민을 했는지는 블랙박스에 가려져 있어. 
우리는 AI 채팅 기록이라는 **'검증 가능한 데이터(Verifiable Data)'**를 통해 그 과정을 추적해.

네가 설계할 루브릭은 다음 세 가지를 실현해야 해:
1. **평가받지 못했던 가치의 발견**: 학생이 AI와 대화하며 범했던 시행착오, 질문을 고쳐 쓴 흔적, AI의 답변을 비판적으로 수용한 지점들을 포착해야 해.
2. **데이터의 증거화**: 단순히 "잘했음"이 아니라, 채팅 기록의 특정 지점을 근거로 내세워 평가의 신뢰도를 확보하는 거야.
3. **피드백의 선순환**: AI의 평가를 보고 교사가 코멘트를 더하고, 학생이 자신의 성장을 확인하는 **삼각 소통(Teacher-AI-Student)**의 시작점이 되어야 해.

---

## 2. 🔍 루브릭 설계의 '딥(Deep)' 가이드

단순히 "질문을 잘했는가?"라고 묻지 마. 더 깊게 파고들어 줘.

### ✅ 측정 가능한 '과정'의 지표들
- **비판적 수용**: AI가 틀린 정보를 줬을 때 이를 잡아내거나, 더 정확한 근거를 요구했는가?
- **질문의 진화**: 답변이 마음에 들지 않을 때, 더 구체적인 제약 조건을 넣어 질문을 정교하게 다듬어갔는가?
- **주도적 응용**: AI의 제안을 그대로 복사하지 않고, 자신의 프로젝트 맥락에 맞게 변형하여 적용하려 노력했는가?
- **메타인지적 성찰**: '학생 자기평가'에 적힌 본인의 의도가 실제 대화 과정에서 얼마나 일관되게 추진되었는가?

## 3. 🛠 기술 사양 및 약속 (JSON Format)

네가 도출한 교육적 통찰은 아래 형식을 통해 우리 시스템에 수혈될 거야.

```json
{
  "name": "루브릭 명칭 (과정 중심 평가의 성격이 드러나게)",
  "description": "어떤 학습 과정의 가치를 증명하고자 하는가?",
  "criteria": [
    {
      "id": "crit_process_1",
      "name": "항목명 (예: 비판적 정보 판별력)",
      "weight": 30,
      "description": "AI 답변의 오류를 점검하거나 신뢰도를 확인하는 과정을 평가",
      "levels": [
        { "score": 5, "description": "AI 대답을 비판적으로 검토하고 구체적인 근거를 추가 요구함" },
        { "score": 3, "description": "대답이 이상할 때 다시 물어보는 수준의 피드백을 보냄" },
        { "score": 1, "description": "AI의 답변을 무비판적으로 수용하고 대화를 종료함" }
      ]
    }
  ]
}
```

## 4. � 파트너 AI에게 전하는 마지막 당부

우리는 **"베리파이어블(Verifiable)한 데이터를 베리파이어블한 평가"**로 바꾸는 마법을 부리고 있어. 

학생은 네 루브릭에 기반한 피드백을 읽으며 "내가 이 대목에서 이렇게 성장했구나"라고 느끼고, 
선생님은 "이 학생의 보이지 않던 노력을 AI 덕분에 발견했네"라고 말할 수 있어야 해.

네가 설계할 루브릭의 글자 하나하나가 학생들에게는 **성장의 이정표**가 될 거야. 
선생님의 교육 철학을 가장 정교한 프롬프트(루브릭)로 번역하는 네 능력을 믿어.

함께 교육의 미래를 그려보자! 🤜🤛
